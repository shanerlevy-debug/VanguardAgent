# VanguardCMA

A Claude Managed Agent (CMA) that runs competitive-intelligence on a schedule, posts findings to Slack, and persists state in Anthropic-hosted memory. **Runs entirely on Anthropic infrastructure** — no AWS, no self-hosted services, no Docker.

## What you get

- A scheduled agent that scans news, blogs, press releases, SEC filings, GitHub, YouTube, Reddit, and job boards for tracked competitors
- Each finding scored (High / Medium / Low) and categorized into one of 12 buckets
- High and Medium findings posted individually to Slack; Low findings batched into a roundup
- An end-of-run summary in Slack so the channel never goes silent
- All state persisted in an Anthropic Memory store — no database to run

## What's in this repo

```
VanguardCMA/
├── vanguard.yaml.example       # the orchestration config you fill in
├── agent.yaml.example          # CMA-native manifest example (regenerated by deploy)
├── secrets.env.example         # API keys (never commit secrets.env)
├── agent/system_prompt.md      # what the agent is told it is
├── skills/                     # 8 skills uploaded to Anthropic at deploy
│   ├── core/                   # framework skills — don't edit unless upgrading
│   └── custom/                 # company-specific skills — edit freely
├── onboard.sh / onboard.ps1    # interactive first-time setup wizard
├── deploy.sh / deploy.ps1      # push to Anthropic (creates venv on first run)
├── scripts/
│   ├── onboard.py              # interactive wizard logic
│   ├── deploy.py               # the deploy logic
│   ├── run_once.py             # fire a single session manually
│   ├── tail.py                 # stream events from a session
│   └── teardown.py             # remove every Anthropic resource
└── docs/                       # step-by-step deployment guide
```

## Quickstart

For full white-glove instructions, follow [`docs/00-overview.md`](docs/00-overview.md). The 30-second version:

```bash
# 1. Interactive onboarding — preflight check + Q&A,
#    writes vanguard.yaml + secrets.env + a stubbed context skill.
./onboard.sh                 # macOS/Linux/Git-Bash
.\onboard.ps1                # Windows PowerShell

# 2. Edit skills/custom/understanding-{your-slug}/SKILL.md to fill in
#    your company context (the highest-leverage step — see docs/04).

# 3. Deploy
./deploy.sh                  # or .\deploy.ps1

# 4. Verify
./run_once.sh     # or .\run_once.ps1 on Windows
```

Then schedule daily runs via a Claude Code routine — see [`docs/07-schedule-via-claude-code.md`](docs/07-schedule-via-claude-code.md).
